# 08 — One-Page NQ1! Trading Plan

This is the daily operating plan for TradingView + QuantVue + Tradovate manual execution.

## 1. Daily objective

Trade only clean NQ/MNQ setups where structure, location, trigger, and risk all agree.

The goal is not to catch every move. The goal is to protect the prop-firm account and take only repeatable A/B setups.

## 2. Pre-market checklist

Before trading:

- Check economic calendar.
- Mark news lockout windows.
- Mark prior day high and low.
- Mark overnight high and low.
- Mark 8am candle zone if using that strategy.
- Mark opening range after the session begins.
- Review 5m SMC structure.
- Decide day type: trend, range, reversal, chop, or no-trade.
- Confirm Tradovate account and bracket order settings.

## 3. Core chart layout

### Bias chart

- NQ1! 5-minute candles
- SMC V2
- Prior highs/lows
- Manual structure period 20
- Order blocks and liquidity

### Execution chart

- NQ1! 1-second traditional Renko
- 4-box Renko for discretionary execution
- Qpro V2
- Moneyball V2
- Qgrid only if adding to a winner

### Context chart

- 15-minute chart
- 15-minute opening range
- FVG / higher-timeframe zones

## 4. Trade qualification

A trade must score at least **7/10** from `02-confluence-system.md`.

Minimum requirements:

- Clear direction from 5m structure or opening range.
- Clear location: zone, OR retest, 8am retest, liquidity sweep, Qwave/Qbands edge, or order block.
- Clear trigger: Qline flip, Qcloud alignment, Moneyball confirmation, or SMC shift.
- Clear stop.
- Clear target.
- No major news risk.

## 5. Primary setups

Trade these first:

1. **15-minute opening range retest**
2. **3-minute opening range breakout/retest**
3. **8am candle retest**
4. **Zone strategy**
5. **SMC liquidity sweep reversal**
6. **Qwave/Qbands reversal to midline**
7. **Qcloud trend continuation**
8. **Qgrid add-to-winner only**

## 6. Entry rules

Before entry:

- Bias agrees with direction.
- Price is at a real level.
- Trigger confirms.
- Stop is known.
- Target is known.
- Reward:risk is acceptable.
- Tradovate bracket is ready.

No entry if:

- Price is in the middle of nowhere.
- Qcloud is chopping.
- Moneyball is noisy.
- Setup is directly into opposing level.
- Stop is too wide for prop-firm risk.
- You are trying to make back a loss.

## 7. Exit rules

Exit or reduce when:

- Target 1 is reached.
- Qline flips against position.
- Price rejects at major level.
- Moneyball momentum clearly reverses.
- Trade thesis is invalidated.
- Daily risk rule requires shutdown.

## 8. Qgrid rule

Qgrid is only for adding to a winner.

Do not use Qgrid to average down.

A valid Qgrid add requires:

- Existing trade is already profitable.
- Pullback is controlled.
- Structure remains valid.
- Qgrid signal appears in original direction.
- Added risk is still inside daily risk plan.

## 9. Prop-firm risk rules

Set these before trading:

- Max daily loss: ________
- Max trades: ________
- Max consecutive losses: ________
- Max contracts: ________
- News lockout window: ________
- Stop trading time: ________

Suggested conservative defaults while testing:

- Stop after 3 losses.
- Stop after 2 full-risk losses.
- Stop after one rule break.
- Use MNQ until 20+ examples prove the setup.

## 10. Journal every trade

Record:

- Setup name
- Direction
- Entry
- Stop
- Target
- Confluence score
- Result in R
- Mistake, if any
- Screenshot/link

If it is not journaled, it does not count as tested.
